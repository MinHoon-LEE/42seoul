################### NET_WHAT #####################

	1. TCP/IP 프로토콜 구성 : 
		1) IP 주소
		2) 서브넷 마스크
		3) 기본 게이트웨이 
	
	1) IP 주소:
		1 -	IPV4 : 32bits
		2 - IPV6 : 128bits

		옥텟(octet) : 8bit 씩 끊어서 읽는것 / 편하게 10진수로 변환해서 192.168.123.132 처럼 읽음

	2) 서브넷 마스크:
		1 - 호스트가 로컬 서브넷에 있는지, 원격 네트워크에 있는지 확인
			ex) 서브넷 마스크 255.255.255.0 -> 처음 24bit 만 네트워크 주소나머지 8bit 호스트 주소
		2 - 인터넷을 관리하는 조직 (InterNIC) 일반 적으로 (A,B,C,D,E) 클래스로 나뉘어짐
			A - 0 ~ 127 사이의 값
			B - 128 ~ 191 사이의 값
			c - 192 ~ 223 사이의 값
			D,E - 연구목적이나 특별한 목적으로 사용
			호스트 주소중
				- 모두(0) -> 네트워크 주소
				- 모두(1) -> 브로드캐스트 주소
			만약에 C 클래스 255.255.255 중 254개 사용가능 
			근데 50 * 3 개의 서버에서 사용하고싶으면
			서브넷 255.255.255.192 해서 62 *4 개로 쪼갠다.
		3 (브로드캐스트 주소)
			1 -	브로드캐스트 주소는 로컬 네트워크 전체에 있는 클라이언트 모두에게 데이터를 보내는 방식
				ARP, DHCP, RIP 등의 프로토콜에 사용된다. (Address 주소중 가장큰 수 이다)
		서브넷팅 - 
			한개의 네트워크를 여러개의 서브넷 네트워크로 분할 하는 것
	3) Private IP (개인 네트워크에 사용)
		CLASS A - 10. ~ 
		CLASS B - 172.16.0.0 ~ 172.31.0.0
		CLASS C - 192.168.0.0 ~ 
	4) CLASS 개념 : 
		하나의 IP 주소에서 네트워크 영역과 호스트영역을 나누는 방법
	5 ) TCP/ UDP :
		TCP : Transmission Control Protocol : 상호 통신 (신뢰성을 보장함)
		UDP : User Datagram Protocol : 일방적 통신 (간단한고 빠른속도로 데이터 전송)

	6) OSI 7계층 model :
		1) Physical : bits
		2) DataLink : Frames
		3) Network : Packets -> IP 로 통신 
		4) Transprot : segments  -> 포트 번호로 통신
		5) session : Data
		6) presentations : Data
		7) Application : Data 
	7) TCP / IP model : 7계층 에서 중요한 TCP 와 IP 만 따와서 사용
		1) Network
		2) Internet 
		3) Transport
		4) Application
	8) DHCP 프로토콜 :
		동적 호스트 설정 프로토콜
		IP 주소, 서브넷 마스크, 게이트웨이 IP주소, DNS IP 주소를 자동으로 일정시간 할당 해주는 인터넷 프로토콜
	9) DNS	프로토콜 :
		도메인 네임과 Ip주소의 대응 관게를 데이터 베이스로 구축해 사용하는 인터넷 프로토콜
		DNS :
			브라우징을 단순화하는 매우 특별한 목적을 수행하는 인터넷상의 또 다른 컴퓨터

		DNS 절차:
			1) 사용자가 브라우제에 URL을 입력한다.
			2) 브라우저가 DNS에 접속하여 입력한 도메인이름과 관련된 IP 주소를 요청
			3) 획득한 IP주소를 사용하여 컴퓨터와 통신 . 특정 페이지 요청
	10) 라우터의 동작원리 :
		라우터는 패킷의 전송경롤ㄹ 결정하기 위해 랜테이블, 네트워크 테이블, 라우팅테이블을 사용한다. 
		네트워크에 연결된 모든 장치들의 주소를 인식하고 이것을 바탕으로 패킷의 전송경로를 결정한다.
		1) 랜테이블 검사를 한다. 이곳에서는 패킷의 목적지가 같은 네트워크에 있는지 아니면 다른네트워크에 있는지 확인
		2) 네트워크테이블을 검사하여 패킷을 전달할 네트워크 주소를 찾아낸다.
		3) 라우팅 테이블을 검새가여 가장적합한 경로를 찾아내서 패킷을 보낸다.
